# 푸드트럭 타이쿤 - 게임 기획서 (GDD)

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 클리커 + 방치형 타이쿤 |
| **테마** | 푸드트럭 운영 |
| **플랫폼** | 모바일 (세로 화면) |
| **엔진** | Unity 3D |
| **카메라** | 고정 (이동/회전 없음) |
| **조작** | 화면 터치 1종류 |

---

## 2. 핵심 루프

```
클릭 → 음식 제작 → 손님에게 판매 → 골드 획득 → 업그레이드 → 반복
                                         ↑
                        자동 생산 (요리사) ─┘
```

---

## 3. 재화

| 재화 | 획득 | 용도 |
|------|------|------|
| **골드** | 클릭 / 자동 생산 | 업그레이드 구매 |

---

## 4. 업그레이드 시스템

### 4-1. 클릭 업그레이드 (3종)

| ID | 이름 | 비용 (Lv1/2/3) | 효과 | 비주얼 |
|----|------|----------------|------|--------|
| `click_revenue` | **클릭 수익** | 100 / 500 / 2000 | 클릭당 골드 배율 x1.5 / x2 / x3 | - |
| `critical_chance` | **크리티컬 확률** | 150 / 600 / 2500 | 크리티컬 발생 확률 10% / 20% / 30% | 여러 FoodPop 생성 |
| `critical_damage` | **크리티컬 데미지** | 200 / 800 / 3000 | 크리티컬 시 메뉴 개수 2 / 3 / 5개 | "x3" 플로팅 텍스트 |

### 4-2. 자동 업그레이드 (3종)

| ID | 이름 | 비용 (Lv1/2/3) | 효과 | 비주얼 |
|----|------|----------------|------|--------|
| `chef_hire` | **요리사 고용** | 100 / 500 / 2000 | 자동 클릭 +1 / +2 / +3명 | 요리사 캐릭터 추가 |
| `cooking_speed` | **요리 속도** | 200 / 800 / 3000 | 자동 클릭 간격 배율 x1.5 / x2 / x3 | - |
| `truck_upgrade` | **트럭 업그레이드** | 300 / 1200 / 5000 | 새 메뉴 해금 + 메뉴 가격 배율 x1.5 / x2 / x3 | 트럭 외관 + 메뉴판 |

### 4-3. 메뉴 시스템

| 메뉴 | 해금 레벨 | 기본 가격 |
|------|----------|----------|
| 핫도그 | 0 (기본) | 10G |
| 타코 | 1 | 15G |
| 버거 | 2 | 30G |
| 피자 | 3 | 50G |

*클릭 시 해금된 메뉴 중 랜덤하게 선택되어 FoodPop 생성*

## 5. 수익 계산 공식

```
[클릭 수익]
일반: (해금된 메뉴 중 랜덤 기본가격 × 메뉴가격배율) × 클릭수익배율
크리티컬: (해금된 메뉴 중 랜덤 기본가격 × 메뉴가격배율) × 클릭수익배율 × 크리티컬데미지(메뉴개수)

[자동 수익 (초당)]
요리사수 × 클릭수익 × 요리속도배율
```

### 계산 예시

```
선택된 메뉴: 버거 (기본 30G)
메뉴가격배율: x1.5 (트럭 Lv1)
클릭수익배율: x1.5 (클릭수익 Lv1)
크리티컬데미지: 3개 (크리티컬데미지 Lv2)
요리사: 2명 (요리사 Lv2)
요리속도: x1.5 (요리속도 Lv1)

메뉴 최종가격: 30 × 1.5 = 45G
일반 클릭: 45 × 1.5 = 67.5G
크리티컬: 45 × 1.5 × 3 = 202.5G (FoodPop 3개 생성)
자동(초당): 2 × 평균클릭수익 × 1.5
```

## 6. 성장 표현

| 단계 | 조건 | 변화 |
|------|------|------|
| **Stage 1** | 시작 | 낡은 트럭 / 핫도그 / 손님 1명 |
| **Stage 2** | 트럭 Lv1 | 깨끗한 트럭 / 타코 해금 / 손님 3명 |
| **Stage 3** | 트럭 Lv2 | 고급 트럭 / 버거 해금 / 손님 5명 |
| **Stage 4** | 트럭 Lv3 | 네온 트럭 / 피자 해금 / 손님 7명 |

## 7. 게임 필 (피드백)

### 7-1. 클릭 피드백

| 요소 | 연출 |
|------|------|
| **음식 팝업** | 해금된 메뉴 중 랜덤 아이콘이 위로 튀어오름 |
| **크리티컬 팝업** | 크리티컬 시 FoodPop이 크리티컬데미지 개수만큼 생성 |
| **크리티컬 텍스트** | "x3" 등 개수 플로팅 텍스트 표시 |
| **골드 표시** | "+67G" 텍스트가 떠오르며 페이드아웃 |
| **트럭 반응** | 살짝 흔들림 (Punch Scale) |
| **파티클** | 연기/김 이펙트 |

### 7-2. 사운드

| 상황 | 효과음 |
|------|--------|
| 클릭 | "치익~" 조리 소리 |
| 골드 획득 | "챠링~" 코인 소리 |
| 크리티컬 | "따단!" 강조음 |
| 업그레이드 | "레벨업!" 효과음 |

## 8. UI 레이아웃

```
┌─────────────────────────┐
│       골드: 12,345      │ ← 보유 골드
│      +5.6G/sec          │ ← 초당 자동 수익
│                         │
│  ┌─────────────────┐    │
│  │ MENU            │    │
│  │ [1] [2] [3] [ ] │    │ ← 메뉴판 (해금 현황)
│  └─────────────────┘    │
│                         │
│        [트럭]           │ ← 푸드트럭 (클릭 영역)
│       [요리사]          │ ← 요리사 (고용 시 표시)
│    [손님] [손님] ...    │ ← 손님 줄
│                         │
├─────────────────────────┤
│ 클릭   │ 크확률│ 크데미지│ ← 클릭 업그레이드
│ Lv2    │ Lv1   │ Lv1    │
├─────────────────────────┤
│ 요리사 │ 속도  │ 트럭   │ ← 자동 업그레이드
│ Lv2    │ Lv1   │ Lv1    │
└─────────────────────────┘
```
